## Components

- **Caddy edge** — the only containerized component (rootless Podman,
  §11.4.76/§11.4.161). Terminates TLS on `:52443` (HTTP `:52080`→HTTPS 301),
  speaks HTTP/3 (QUIC), compresses with Brotli (custom xcaddy build), gates every
  request via `forward_auth` → `helix-auth`, blocks `/proxy`, strips client
  `X-Helix-User`. `deploy/Caddyfile`, `deploy/caddy/Dockerfile`.
- **helix-auth** — host-native **Gin** forward-auth gate (`systemd --user`,
  loopback `127.0.0.1:8081`, **non-root**). Issues + verifies the SSH-key
  challenge (`ssh-keygen -Y verify` vs the account's `authorized_keys`), issues
  the signed session cookie, **fails closed**, stores no password. `services/auth_gate/`.
- **code-server** — browser VS Code, **host-native as the real user** (`milosvasic`),
  `--auth none` on loopback `127.0.0.1:8080` (auth enforced upstream at the gate/edge),
  projects served read-write, Explorer defaults to `$PROJECTS_ROOT`.
  `deploy/systemd/helix-code-server.service`.
- **Operator scripts** — install / setup / start / stop / status / doctor +
  `install-auth.sh` (tarball code-server + units), `install-edge-boot.sh` (edge
  Quadlet), `harden-loopback.sh` (UID-scoped loopback firewall).
- **Boot-survival** — rootless `systemd --user` units (linger) + the edge Quadlet.

## Honest boundaries (§11.4.6)

- **No §11.4.153 real-use VIDEO exists yet** for any HelixCode feature — every row's
  **Video-confirmation is PENDING**. The autonomous captured evidence backing each
  PASS is the `qa-results/**` log + `.txt` artefact cited in the row (§11.4.69/§11.4.107),
  NOT a recorded video. Producing the per-feature real-use videos is a tracked gap;
  nothing here is claimed video-confirmed until the recording lands.
- **Editor "jail" is cosmetic.** The Explorer defaults to `$PROJECTS_ROOT`, but the
  integrated terminal, **File > Open Folder**, and extensions retain **full real-user
  (`milosvasic`) host access by design** — a convenience view, not a security
  boundary. Real isolation would need a container / VM / chroot (out of scope per
  operator decision). Defense-in-depth (deploy stream): `--disable-workspace-trust`
  + `/proxy/` blocked at the edge.
- **`--auth none` on loopback:8080 residual.** code-server runs unauthed on loopback;
  all auth is at the gate/edge. A local process reaching `127.0.0.1:8080` bypasses the
  gate — mitigated by `scripts/harden-loopback.sh` (a UID-scoped firewall rule the
  operator applies as root; the project never uses sudo).
- **Full in-browser editor interaction** (typing, file tree, command palette) is
  **OPERATOR-ATTENDED** until a browser-automation adapter lands (HCA-QA-UI-001);
  every HTTP/CLI-drivable journey is autonomously verified.
- **Post-`0.0.3` additions now PASS (were in-flight/PENDING at Revision 2):** the
  login-form clipboard copy/paste buttons (`login_enhance.js`, commit `8351e4c` — LIVE
  3/3 incl §11.4.170 CDP pixel proof), the VS Code Dark default theme (`97a8c69` — config
  3/3 + rendered-pixel proof), the popular-extensions matrix (`97a8c69`), the UI-driven
  marketplace journey (`8351e4c` — browse→search→Install-click, operator_attended on
  install-completion), and the rootless-Quadlet edge (now LIVE, `active (running)`).
